44 // AUDIO_BLOCK_SAMPLES determines how many samples the audio library processes
45 // per update. It may be reduced to achieve lower latency response to events,
46 // at the expense of higher interrupt and DMA setup overhead.
47 //
48 // Less than 32 may not work with some input & output objects. Multiples of 16
49 // should be used, since some synthesis objects generate 16 samples per loop.
50 //
51 // Some parts of the audio library may have hard-coded dependency on 128 samples.
52 // Please report these on the forum with reproducible test cases. The following
53 // audio classes are known to have problems with smaller block sizes:
54 // AudioInputUSB, AudioOutputUSB, AudioPlaySdWav, AudioAnalyzeFFT256,
55 // AudioAnalyzeFFT1024
56

58 #define AUDIO_BLOCK_SAMPLES 128
59 #endif
